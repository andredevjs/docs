# Token Data Model

The `TokenEntity` is the core data structure representing tokens on the Odin platform, containing comprehensive information about token metadata, market data, trading statistics, and social features.

## Complete Schema

<ResponseField name="TokenEntity" type="object">
The complete token object with all available fields and nested structures.
</ResponseField>

### Basic Information

<ResponseField name="id" type="string" required>
Unique token identifier used across all API endpoints.
</ResponseField>

<ResponseField name="name" type="string" required>
Human-readable token name displayed in user interfaces.
</ResponseField>

<ResponseField name="description" type="string">
Detailed description of the token's purpose and functionality.
</ResponseField>

<ResponseField name="image" type="string">
URL to the token's logo or representative image.
</ResponseField>

<ResponseField name="creator" type="string" required>
Principal ID of the token creator/deployer.
</ResponseField>

<ResponseField name="ticker" type="string" required>
Short symbol/ticker used for trading (e.g., "BTC", "ETH").
</ResponseField>

<ResponseField name="created_time" type="string" required>
ISO 8601 timestamp of when the token was created.
</ResponseField>

### Market Data

<ResponseField name="price" type="number" required>
Current token price in the base currency (typically BTC satoshis).
</ResponseField>

<ResponseField name="marketcap" type="number" required>
Current market capitalization calculated as price × circulating supply.
</ResponseField>

<ResponseField name="volume" type="number" required>
24-hour trading volume in base currency.
</ResponseField>

<ResponseField name="holder_count" type="number" required>
Total number of unique token holders.
</ResponseField>

### Price Analytics

<ResponseField name="price_5m" type="integer">
Token price from 5 minutes ago (for percentage change calculations).
</ResponseField>

<ResponseField name="price_1h" type="integer">
Token price from 1 hour ago.
</ResponseField>

<ResponseField name="price_6h" type="integer">
Token price from 6 hours ago.
</ResponseField>

<ResponseField name="price_1d" type="integer">
Token price from 24 hours ago.
</ResponseField>

### Status Flags

<ResponseField name="bonded" type="boolean" required>
Whether the token is still in the bonding curve phase (`true`) or has graduated to full DEX trading (`false`).
</ResponseField>

<ResponseField name="featured" type="boolean" required>
Whether the token has featured status for enhanced visibility.
</ResponseField>

<ResponseField name="verified" type="boolean" required>
Whether the token has been verified by the platform.
</ResponseField>

<ResponseField name="external" type="boolean" required>
Whether the token is imported from an external source.
</ResponseField>

### Trading Configuration

<ResponseField name="trading" type="boolean" required>
Whether trading is enabled for this token.
</ResponseField>

<ResponseField name="deposits" type="boolean" required>
Whether deposits are enabled for this token.
</ResponseField>

<ResponseField name="withdrawals" type="boolean" required>
Whether withdrawals are enabled for this token.
</ResponseField>

### Social Features

<ResponseField name="comment_count" type="number" required>
Total number of comments on this token.
</ResponseField>

<ResponseField name="last_comment_time" type="string">
ISO 8601 timestamp of the most recent comment.
</ResponseField>

<ResponseField name="twitter" type="string">
Twitter handle or URL associated with the token.
</ResponseField>

<ResponseField name="website" type="string">
Official website URL for the token.
</ResponseField>

<ResponseField name="telegram" type="string">
Telegram group or channel for the token community.
</ResponseField>

<ResponseField name="twitter_verified" type="boolean" required>
Whether the associated Twitter account has been verified.
</ResponseField>

### Trading Statistics

<ResponseField name="buy_count" type="number" required>
Total number of buy transactions for this token.
</ResponseField>

<ResponseField name="sell_count" type="number" required>
Total number of sell transactions for this token.
</ResponseField>

<ResponseField name="sold" type="number" required>
Total amount of tokens sold.
</ResponseField>

<ResponseField name="txn_count" type="integer" required>
Total number of transactions (buys + sells).
</ResponseField>

### Liquidity Information

<ResponseField name="btc_liquidity" type="integer" required>
Total BTC liquidity in the token's pools.
</ResponseField>

<ResponseField name="user_btc_liquidity" type="integer" required>
User's BTC liquidity contribution (when requesting user-specific data).
</ResponseField>

<ResponseField name="user_token_liquidity" type="number" required>
User's token liquidity contribution.
</ResponseField>

<ResponseField name="user_lp_tokens" type="number" required>
User's LP (Liquidity Provider) token balance.
</ResponseField>

<ResponseField name="token_liquidity" type="number" required>
Total token liquidity in pools.
</ResponseField>

<ResponseField name="liquidity_threshold" type="integer" required>
Threshold for graduating from bonding curve to full DEX.
</ResponseField>

### Supply Information

<ResponseField name="total_supply" type="number" required>
Total token supply (minted amount).
</ResponseField>

<ResponseField name="threshold" type="number" required>
Progress threshold for bonding curve graduation.
</ResponseField>

<ResponseField name="progress" type="number" required>
Current progress toward graduation (0-100%).
</ResponseField>

### Trading Fees

<ResponseField name="swap_fees" type="integer" required>
Total fees collected from swaps (all-time).
</ResponseField>

<ResponseField name="swap_volume" type="integer" required>
Total swap volume (all-time).
</ResponseField>

<ResponseField name="swap_fees_24" type="integer" required>
Fees collected from swaps in the last 24 hours.
</ResponseField>

<ResponseField name="swap_volume_24" type="integer" required>
Swap volume in the last 24 hours.
</ResponseField>

### Technical Details

<ResponseField name="decimals" type="number" required>
Number of decimal places for token precision.
</ResponseField>

<ResponseField name="divisibility" type="number" required>
Token divisibility (for Bitcoin Runes).
</ResponseField>

<ResponseField name="icrc_ledger" type="string">
ICRC ledger canister ID (for IC-based tokens).
</ResponseField>

### Bitcoin Runes Integration

<ResponseField name="rune" type="string">
Bitcoin Rune name (if applicable).
</ResponseField>

<ResponseField name="rune_id" type="string" required>
Bitcoin Rune ID for integration with Runes protocol.
</ResponseField>

### Holder Analytics

<ResponseField name="holder_dev" type="number" required>
Percentage of supply held by developers/team.
</ResponseField>

<ResponseField name="holder_top" type="number" required>
Percentage of supply held by top holders.
</ResponseField>

### Activity Tracking

<ResponseField name="last_action_time" type="string" required>
ISO 8601 timestamp of the last trading or interaction activity.
</ResponseField>

### Relationships

<ResponseField name="rel_ledgers" type="array">
Array of related ledger entries showing user balances.

<Expandable title="LedgerEntity Structure">
<ResponseField name="user" type="string" required>
Principal ID of the token holder.
</ResponseField>

<ResponseField name="token" type="string" required>
Token ID.
</ResponseField>

<ResponseField name="balance" type="number" required>
User's token balance.
</ResponseField>

<ResponseField name="user_username" type="string" required>
Display username of the holder.
</ResponseField>

<ResponseField name="user_image" type="string" required>
Profile image URL of the holder.
</ResponseField>
</Expandable>
</ResponseField>

<ResponseField name="rel_tags" type="array">
Array of tags associated with this token.

<Expandable title="TagEntity Structure">
<ResponseField name="id" type="number" required>
Unique tag identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
Tag name (e.g., "defi", "gaming").
</ResponseField>

<ResponseField name="description" type="string">
Tag description explaining its purpose.
</ResponseField>

<ResponseField name="created_time" type="string" required>
When the tag was created.
</ResponseField>
</Expandable>
</ResponseField>

## Field Relationships

<AccordionGroup>
<Accordion title="Price Calculation Fields">
  **Related Fields:**
  - `price` - Current price
  - `price_5m`, `price_1h`, `price_6h`, `price_1d` - Historical prices
  - `marketcap` = `price` × `total_supply`
  - `volume` - 24-hour trading volume
  
  **Usage:** Calculate price changes and market metrics.
</Accordion>

<Accordion title="Liquidity Fields">
  **Related Fields:**
  - `btc_liquidity` - Total BTC in pools
  - `token_liquidity` - Total tokens in pools
  - `user_btc_liquidity` - User's BTC contribution
  - `user_token_liquidity` - User's token contribution
  - `user_lp_tokens` - User's LP token balance
  
  **Usage:** Track liquidity provision and calculate yields.
</Accordion>

<Accordion title="Trading Activity">
  **Related Fields:**
  - `buy_count` + `sell_count` = `txn_count`
  - `swap_fees` vs `swap_fees_24` (all-time vs 24h)
  - `swap_volume` vs `swap_volume_24` (all-time vs 24h)
  - `last_action_time` - Most recent activity
  
  **Usage:** Analyze trading patterns and activity levels.
</Accordion>

<Accordion title="Bonding Curve Progress">
  **Related Fields:**
  - `bonded` - Whether still in bonding phase
  - `threshold` - Target for graduation
  - `progress` - Current progress percentage
  - `liquidity_threshold` - BTC threshold for graduation
  
  **Calculation:** `progress = (current_liquidity / threshold) * 100`
</Accordion>
</AccordionGroup>

# Token Discovery API

The token discovery endpoint provides comprehensive tools for browsing, searching, and filtering tokens on the Odin platform with advanced sorting and pagination capabilities.

## Get All Tokens

<ParamField path="GET /tokens" type="endpoint">
Retrieve a paginated list of tokens with powerful filtering and sorting options.
</ParamField>

### Query Parameters

#### Pagination

<ParamField query="page" type="number" default="1">
Page number for pagination. Must be >= 1.
</ParamField>

<ParamField query="limit" type="number" default="10">
Number of tokens per page. Range: 1-100.
</ParamField>

#### Environment Filter

<ParamField query="env" type="string">
Filter by deployment environment.

<Expandable title="Environment Options">
- `production` - Live production tokens
- `staging` - Staging environment tokens  
- `development` - Development tokens
</Expandable>
</ParamField>

#### Sorting Options

<ParamField query="sort" type="string">
Sort tokens by various metrics in ascending or descending order.

<Expandable title="Sort Options">
- `marketcap:asc` / `marketcap:desc` - Market capitalization
- `price:asc` / `price:desc` - Token price
- `holder_count:asc` / `holder_count:desc` - Number of holders
- `created_time:asc` / `created_time:desc` - Creation date
- `name:asc` / `name:desc` - Alphabetical by name
- `last_comment_time:asc` / `last_comment_time:desc` - Recent activity
- `comment_count:asc` / `comment_count:desc` - Comment count
- `volume:asc` / `volume:desc` - Trading volume
- `sold:asc` / `sold:desc` - Tokens sold
- `txn_count:asc` / `txn_count:desc` - Transaction count
- `btc_liquidity:asc` / `btc_liquidity:desc` - BTC liquidity
- `swap_fees:asc` / `swap_fees:desc` - Swap fees collected
- `swap_volume:asc` / `swap_volume:desc` - Swap volume
- `last_action_time:asc` / `last_action_time:desc` - Last activity
- `price_delta_5m:asc` / `price_delta_5m:desc` - 5-minute price change
- `price_delta_1h:asc` / `price_delta_1h:desc` - 1-hour price change
- `price_delta_6h:asc` / `price_delta_6h:desc` - 6-hour price change
- `price_delta_1d:asc` / `price_delta_1d:desc` - 24-hour price change
- `ascension:asc` / `ascension:desc` - Ascension ranking
</Expandable>
</ParamField>

#### Status Filters

<ParamField query="bonded" type="boolean">
Filter by bonding curve status. `true` for tokens still in bonding phase, `false` for graduated tokens.
</ParamField>

<ParamField query="featured" type="boolean">
Filter for featured tokens with enhanced visibility.
</ParamField>

<ParamField query="etched" type="boolean">
Filter by etched status (Bitcoin Runes specific).
</ParamField>

<ParamField query="verified" type="boolean">
Filter for verified tokens with confirmed authenticity.
</ParamField>

<ParamField query="twitter_verified" type="boolean">
Filter for tokens with verified Twitter accounts.
</ParamField>

<ParamField query="ascended" type="boolean">
Filter for tokens that have ascended to full DEX trading.
</ParamField>

#### Numeric Range Filters

<ParamField query="price_min" type="number">
Minimum token price filter.
</ParamField>

<ParamField query="price_max" type="number">
Maximum token price filter.
</ParamField>

<ParamField query="volume_min" type="number">
Minimum 24-hour trading volume.
</ParamField>

<ParamField query="volume_max" type="number">
Maximum 24-hour trading volume.
</ParamField>

<ParamField query="holders_min" type="number">
Minimum number of token holders.
</ParamField>

<ParamField query="holders_max" type="number">
Maximum number of token holders.
</ParamField>

<ParamField query="marketcap_min" type="number">
Minimum market capitalization.
</ParamField>

<ParamField query="marketcap_max" type="number">
Maximum market capitalization.
</ParamField>

<ParamField query="comment_count_min" type="number">
Minimum number of comments.
</ParamField>

<ParamField query="comment_count_max" type="number">
Maximum number of comments.
</ParamField>

#### Social Media Filters

<ParamField query="has_twitter" type="boolean">
Filter for tokens with Twitter accounts.
</ParamField>

<ParamField query="has_telegram" type="boolean">
Filter for tokens with Telegram groups.
</ParamField>

<ParamField query="has_website" type="boolean">
Filter for tokens with official websites.
</ParamField>

#### Search and Identification

<ParamField query="search" type="string">
Full-text search across token fields: id, name, description, rune, creator, telegram, website, twitter, ticker.
</ParamField>

<ParamField query="id_in" type="array">
Filter by specific token IDs (comma-separated).
</ParamField>

<ParamField query="id_not_in" type="array">
Exclude specific token IDs (comma-separated).
</ParamField>

<ParamField query="creator" type="string">
Filter by token creator's principal ID.
</ParamField>

<ParamField query="favorite" type="string">
Filter for tokens favorited by a specific user.
</ParamField>

#### Tag Filters

<ParamField query="include_tags" type="string">
Include tokens with specific tags (comma-separated tag names).
</ParamField>

<ParamField query="exclude_tags" type="string">
Exclude tokens with specific tags (comma-separated tag names).
</ParamField>

#### Token Type Filter

<ParamField query="type_in" type="string">
Filter by token type.

<Expandable title="Token Types">
- `internal` - Internal platform tokens
- `token` - Standard user-created tokens  
- `currency` - Currency/stablecoin tokens
</Expandable>
</ParamField>

## Code Examples

<CodeGroup>
```javascript Node.js
// Basic token discovery
const response = await fetch('https://api.odin.com/tokens?limit=20&sort=marketcap:desc');
const { data: tokens, count, page, limit } = await response.json();

// Advanced filtering
const filteredTokens = await fetch(
  'https://api.odin.com/tokens?' + new URLSearchParams({
    featured: 'true',
    verified: 'true',
    marketcap_min: '100000',
    has_twitter: 'true',
    sort: 'volume:desc',
    limit: '50'
  })
);

// Search functionality
const searchResults = await fetch(
  'https://api.odin.com/tokens?search=bitcoin+defi&sort=relevance:desc'
);
```

```python Python
import requests

# Basic token discovery
response = requests.get(
    'https://api.odin.com/tokens',
    params={
        'limit': 20,
        'sort': 'marketcap:desc'
    }
)
tokens_data = response.json()

# Advanced filtering
filtered_response = requests.get(
    'https://api.odin.com/tokens',
    params={
        'featured': True,
        'verified': True,
        'marketcap_min': 100000,
        'has_twitter': True,
        'sort': 'volume:desc',
        'limit': 50
    }
)

# Search with multiple criteria
search_response = requests.get(
    'https://api.odin.com/tokens',
    params={
        'search': 'bitcoin defi',
        'bonded': False,
        'price_min': 1,
        'price_max': 1000
    }
)
```

```curl cURL
# Get top tokens by market cap
curl "https://api.odin.com/tokens?sort=marketcap:desc&limit=10"

# Search for DeFi tokens
curl "https://api.odin.com/tokens?search=defi&include_tags=defi&verified=true"

# Filter high-volume tokens
curl "https://api.odin.com/tokens?volume_min=100000&sort=volume:desc&limit=25"

# Get tokens from specific creator
curl "https://api.odin.com/tokens?creator=rdmx6-jaaaa-aaaah-qcaiq-cai&sort=created_time:desc"
```

## Common Use Cases

<AccordionGroup>
<Accordion title="Token Marketplace">
  Create a comprehensive token marketplace with sorting and filtering:
  
  ```javascript
  // Popular tokens (high market cap + verified)
  const popularTokens = await fetch(
    '/tokens?verified=true&marketcap_min=50000&sort=marketcap:desc&limit=20'
  );
  
  // Trending tokens (recent activity)
  const trendingTokens = await fetch(
    '/tokens?sort=last_action_time:desc&volume_min=1000&limit=15'
  );
  
  // New launches
  const newTokens = await fetch(
    '/tokens?sort=created_time:desc&limit=10'
  );
  ```
</Accordion>

<Accordion title="Investment Screening">
  Screen tokens based on investment criteria:
  
  ```javascript
  // High-quality tokens for investment

    '/tokens?' + new URLSearchParams({

      marketcap_min: '100000',

      has_website: 'true',
      sort: 'volume:desc'
  ```

  ```javascript
    '/tokens?has_telegram=true&comment_count_min=10&sort=last_comment_time:desc'
  ```

</Accordion>
